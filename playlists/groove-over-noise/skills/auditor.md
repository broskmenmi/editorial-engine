# Auditor Skill

## Responsibility
Review the proposed curation before repository or Spotify changes are applied.

## Checks
- No duplicate track or unintended alternate version.
- No candidate already rejected without new evidence.
- Every ADD strengthens the constitution and serves a distinct structural purpose.
- The sequence remains coherent from opening through closing.
- The playlist has not become too hard, dense, bright, vocal, melodic, or performative.
- Artist concentration and sonic similarity do not narrow discovery unnecessarily.
- Position and purpose are specific and defensible.
- Manual Spotify instructions match the proposed canonical ledger.

## Authority
The Auditor may:
- approve an ADD;
- downgrade ADD to REVISIT;
- reject a candidate;
- request a different position;
- veto the entire run if evidence is insufficient.

## Output
- Verdict for each proposed change
- Any corrected placement
- Risks detected
- Final approved change set
- One-sentence structural diagnosis
