# GROOVE OVER NOISE — Automation Runbook

The GitHub repository is the persistent source of truth.

## Read before every run

1. `constitution.md`
2. `ledger.md`
3. `discoveries.md`
4. `rejected.md`
5. `notes.md`

## Curation procedure

1. Find exactly three candidates.
2. Classify each as **ADD**, **REVISIT**, or **REJECT**.
3. For every ADD, record:
   - Track
   - Artist
   - Position
   - Purpose
   - Reason
4. Position must use a concrete ledger anchor and a structural role whenever possible.
5. Apply all ADD decisions to `ledger.md` in their intended order.
6. Append the complete run to `discoveries.md`.
7. Add rejected tracks to `rejected.md` with reasons.
8. Update `notes.md` with the playlist's current structural need.
9. Sync Spotify only when playlist-editing capability is available.
10. Never create a duplicate Spotify playlist unless explicitly instructed.

## Playback rule

The canonical ledger is designed for ordered playback. Spotify Mix is suitable for discovery but may break the intended editorial arc.
